# 🎵 **Music Store Data Analysis (SQL)**

## 📌 **Project Overview**
This project features a comprehensive data analysis of a digital music store database using SQL. By writing optimized relational database queries, this project extracts actionable business intelligence regarding sales performance, customer demographics, and product popularity to help drive strategic business growth.

## 🛠️ **Tech Stack & Tools Used**
* *Database/Language:* SQL (PostgreSQL / MySQL / SQL Server)
* *Advanced Concepts Used:* Multi-table JOINs, Common Table Expressions (CTEs), Window Functions, Data Aggregations, and Nested Subqueries.
* *Platform:* GitHub (Version Control & Portfolio Showcase)


## 📁 **Repository Structure**
* Music_Store_Query.sql - Main script containing all analytical SQL queries.
* Music_Store_database.sql - Primary backup script to recreate the entire database environment.
* Music Store Analysis-Questions.pdf - Document listing all the business problem statements solved in this project.
* schema_diagram.png - Visual map of the relational database schema.


## 📊 **Key Business Questions Answered**
The analysis directly addresses several core operational and financial questions:
1. Identifying senior- Most employees and mapping overall organizational performance.
2. Ranking top revenue- Generating geographic regions and countries based on total invoice value.
3. Isolating high- Value transactions and customer spending habits.
4. Pinpointing the store’s highest- Value customer to enable targeted loyalty marketing.
5. Analyzing genre- Specific performance (e.g., identifying top Rock music listeners and tracks).
